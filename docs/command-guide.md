# QQ Bot Command Guide

本文档记录当前 QQ bot 已实现的稳定命令。除特别说明外，私聊可直接发送；群聊中需要 @ bot 或回复 bot 的消息才会处理。

## 通用规则

- 私聊：默认会响应普通聊天和命令。
- 群聊：默认只响应 @ bot 或回复 bot 的消息。
- Owner-only：只有 `.env` 中 `OWNER_QQ_IDS` 配置的主人账号可用。
- Private-only：只能私聊使用，避免在群里暴露隐私、联系人、群号、风格样本或草稿。
- 运行数据默认写入 `data/`，不会提交到 Git。

## 基础帮助与权限

### `/help`

显示基础命令列表。

用法：

```text
/help
```

### `/tools`

查看当前可用工具。普通用户只看到低风险工具；主人会看到管理工具。

别名：

```text
/tools
/工具
工具
工具列表
```

### `/权限`

查看当前用户权限状态，包括是否为主人、是否在信任用户名单中。

别名：

```text
/权限
/owner
权限
我的权限
```

## 普通聊天与会话

### 普通聊天

直接发送自然语言即可。bot 会结合短期会话、机器人 persona、当前用户画像回复。

群聊中必须 @ bot 或回复 bot。

### `/clear`

清空当前会话历史。

权限：

- 私聊：当前用户可清空自己的当前会话。
- 群聊：Owner-only，并需要 `/确认 <id>` 二次确认。

别名：

```text
/clear
清空历史
清除记忆
```

群聊中发送 `/clear` 后会返回确认 ID：

```text
/确认 <id>
/取消 <id>
```

## 确认与取消

### `/确认`

执行待确认操作。当前用于群聊清空、白名单变更、清空风格样本等高风险动作。

权限：Owner-only。

用法：

```text
/确认
/确认 <id>
确认 <id>
/confirm <id>
```

说明：

- 不带 ID 时列出当前用户的待确认操作。
- 待确认操作默认 10 分钟过期。
- 执行结果会写入 `data/action_logs.jsonl`。

### `/取消`

取消待确认操作。

权限：Owner-only。

用法：

```text
/取消
/取消 <id>
取消 <id>
/cancel <id>
```

说明：

- 不带 ID 时列出当前用户的待确认操作。
- 取消结果会写入 `data/action_logs.jsonl`。

## 用户资料记忆

### `记住：...`

显式保存当前用户资料。适合保存称呼、偏好、长期背景等。

用法：

```text
记住：我喜欢简洁直接的回答
记住：称呼=小付
/remember 我是一名程序员
```

### `忘记：...`

删除当前用户资料。

用法：

```text
忘记：偏好
忘记我的资料
清除我的资料
删除我的资料
/forget 偏好
/forget
```

### `我的资料`

查看 bot 记住的当前用户资料。

别名：

```text
我的资料
我的记忆
你记住了什么
/profile
```

### `记忆查询`

搜索当前用户画像中已保存的资料。

用法：

```text
记忆查询 关键词
/memory search 关键词
/memory 查询 关键词
```

### `记忆开关`

控制自动事实抽取。显式 `记住：...` 不受影响。

权限：Owner-only。

用法：

```text
记忆开关
记忆开关 开
记忆开关 关
/memory on
/memory off
```

## 工具命令

### `时间`

查看本机当前时间。

别名：

```text
时间
现在几点
/time
/时间
```

### `计算`

执行安全数学计算。只支持数字、括号、`+ - * / // % **`，不支持函数、变量、导入、属性访问。

用法：

```text
计算：1 + 2 * 3
计算 (10 + 2) / 3
/calc 2 ** 8
/calculate 100 / 4
```

### `待办`

管理当前 QQ 用户自己的待办。

用法：

```text
待办
待办 添加 买牛奶
待办 新增 明天整理资料
待办 完成 1
待办 完成 <待办ID前缀>
/todo
/todo add 买牛奶
/todo done 1
/待办 列表
```

说明：

- `待办` 或 `待办 列表` 查看未完成待办。
- `待办 添加 ...` 新增待办。
- `待办 完成 1` 按列表序号完成待办。
- `待办 完成 <ID>` 按待办 ID 前缀完成待办。

## 信任名单

信任名单是未来自动代聊和高风险工具的权限基座。当前阶段只记录名单，不改变普通聊天默认行为。

权限：Owner-only、Private-only。

白名单新增和删除都会先生成待确认操作，需要再发送 `/确认 <id>` 才会真正写入。

### 查看名单

```text
/白名单
白名单
信任名单
/access
```

### 添加或删除信任用户

```text
/白名单 添加用户 <QQ> [备注]
/白名单 删除用户 <QQ>
```

示例：

```text
/白名单 添加用户 123456789 老朋友
/白名单 删除用户 123456789
```

### 添加或删除信任群

```text
/白名单 添加群 <群号> [备注]
/白名单 删除群 <群号>
```

示例：

```text
/白名单 添加群 987654321 测试群
/白名单 删除群 987654321
```

## 风格画像与草稿

风格画像用于学习主人的说话方式。风格命令当前全部是 Owner-only、Private-only。

### `/风格 查看`

查看当前风格画像摘要，包括语气、长度、表情习惯、样本数、导入来源和蒸馏统计。

```text
/风格 查看
```

### `/风格 设置`

手动设置风格字段。

可设置字段：

- `语气`
- `长度`
- `表情`
- `习惯`
- `避免`

用法：

```text
/风格 设置 语气=自然、简短、像我本人
/风格 设置 习惯=短句；少解释；必要时用一点表情
/风格 设置 避免=编造事实；替主人承诺行动
```

### `/风格 导入`

手动导入一小段主人真实回复样本。这个样本会作为可选示例进入风格 prompt。

用法：

```text
/风格 导入 在的在的，刚看到，我来处理
```

说明：

- 单条样本有长度限制。
- 疑似包含 token、密码、密钥等敏感内容会被拒绝保存。

### `/风格 导入文件`

导入聊天记录文件并生成预览。文件必须先放到：

```text
data/style_profiles/import_inbox/
```

支持格式：

- `.txt`
- `.json`
- `.csv`

用法：

```text
/风格 导入文件 <文件名> 我=<你的昵称或QQ>
```

示例：

```text
/风格 导入文件 最近联系人.txt 我=36
/风格 导入文件 chat.csv 我=你的昵称,你的QQ号
```

说明：

- 第一步只生成预览和 `import_id`。
- 预览会统计解析记录数、你的消息数、跳过敏感消息数、平均长度和常见表达。
- 不会保存对方原话到画像。
- 不会写入普通用户记忆。

### `/风格 确认导入`

确认一次文件导入预览，并把蒸馏摘要写入风格画像。

用法：

```text
/风格 确认导入 <import_id>
```

示例：

```text
/风格 确认导入 20260508_025210_8a717014da
```

### `/风格 蒸馏`

从已保存的直接样本重新生成风格画像摘要。

用法：

```text
/风格 蒸馏
```

说明：

- 当前主要用于直接 `/风格 导入 <sample>` 的小样本重建。
- 批量文件导入已经在确认导入时写入本地蒸馏摘要。

### `/风格 离线蒸馏`

从 QCE JSON 导出目录运行 Stage 5B 离线蒸馏，生成风格摘要和高质量样本索引。

用法：

```text
/风格 离线蒸馏
/风格 离线蒸馏 <QCE_JSON目录>
```

说明：

- Owner-only、Private-only。
- 默认输入目录会从配置的 QCE 导出根目录中选择最近的 JSON 批次。
- 可通过环境变量 `QQBOT_STYLE_QCE_INPUT_DIR`、`QQBOT_STYLE_EXPORT_ROOT`、`QQBOT_STYLE_SELF_UIN` 覆盖默认输入目录、导出根目录和本人 QQ。
- 输出目录默认在对应导出批次的 `distill-runs/` 下。
- 摘要和样本索引不保存聊天正文。
- 样本索引只保存 `source_file_id`、记录序号、消息 ID hash、时间桶、长度、元素类型、质量分等元数据。
- 为本地复现保留 `source_catalog_private.json`，其中只映射源文件，不保存聊天正文。
- 命令执行可能较慢，建议长批次由 Codex 旁路监控 QCE/NapCat 状态。

### `/风格 评估`

查看最近一次 Stage 5B 离线蒸馏的就绪度、样本量、关系来源数量和风险建议。

用法：

```text
/风格 评估
```

说明：

- Owner-only、Private-only。
- 只读取 `evaluation_report.json`。
- 不显示历史上下文、真实回复正文、联系人、群名或 QQ。

### `/风格 关系`

查看关系/场景来源画像摘要，例如私聊/群聊数量、强样本源数量、低证据来源数量和标签分布。

用法：

```text
/风格 关系
```

说明：

- Owner-only、Private-only。
- 只显示 `source_file_id` 和统计标签，不显示联系人、群名、QQ 或聊天正文。

### `/风格 场景`

查看场景画像摘要，例如群聊插话、私聊短回复、显式回复、问句回复等场景分布。

用法：

```text
/风格 场景
```

说明：

- Owner-only、Private-only。
- 只显示场景标签、计数、平均长度和推荐回复形态。
- 不显示历史上下文或真实回复正文。

### `/风格 检索`

用当前对方消息在本地历史索引中检索相似样本。

用法：

```text
/风格 检索 你看下这个安排行不行
```

说明：

- Owner-only、Private-only。
- 会临时读取本地 QCE JSON 计算相似度，但不会返回或保存历史聊天正文。
- 返回结果只包含 `sample_id/source_file_id`、相似度、质量分、回复长度和上下文数量。
- 当前用于调试检索层，还不会把历史原文发送给模型生成回复。

### `/风格 清空样本`

清空手动导入的小样本。

用法：

```text
/风格 清空样本
```

说明：

- 会生成待确认操作，需要再发送 `/确认 <id>`。
- 当前只清空直接样本，不等同于删除所有历史导入来源。

### `/用我的风格回复`

按当前风格画像生成回复草稿，不会代替主人自动发送。若已有 Stage 5B 离线蒸馏结果，会自动接入相似样本索引、关系画像和场景画像的统计元数据来生成草稿。

用法：

```text
/用我的风格回复：你看下这个安排行不行
用我的风格回复：这个怎么回比较自然
风格回复：对方消息
/style draft 对方消息
```

说明：

- 只返回草稿正文。
- 不会自动发送给对方。
- 会读取 Stage 5B 的相似样本索引、关系/来源标签、场景建议和长度统计，形成“生成闭环 v1”。
- 当前闭环不会把历史聊天原文发送给模型，只使用统计和索引元数据；真实历史原句 few-shot 需要后续单独授权和审计。
- 遇到主人当前状态、位置、是否有空、是否完成某事等未知事实时，会尽量生成不确认事实的自然草稿。

## Legacy Agent Mode

当前 `AGENT_MODE = False`，旧 Agent Mode 未启用。

如果未来启用旧 Agent Mode，存在 `/tasks` 查看任务列表的入口，但该路径尚未重构，不建议生产使用。后续计划会用受控 Agent Mode 替代。

## 后续路线

### Stage 5B: 蒸馏质量、评估与生成闭环

已有第一版可用链路：

- QCE JSON 离线蒸馏。
- 全局风格摘要。
- 关系/来源画像。
- 场景画像。
- 相似样本索引和检索。
- `/用我的风格回复` 自动接入 Stage 5B 元数据生成草稿。

继续计划：

- 重构更细的 style profile 字段。
- 继续优化“对方上一句/上下文 -> 主人真实回复”的样本索引。
- `/风格 评估`、`/风格 关系`、`/风格 场景`、`/风格 检索` 已有第一版。
- 优化常见表达提取，去掉泛词。
- 将真实联系人/群上下文映射到对应关系画像。
- 如需把真实历史原句作为 few-shot 示例发送给模型，需要增加 owner 显式授权、审计日志和可关闭开关。

### Stage 7: 受控 Agent Mode

计划：

- 替换旧 `AGENT_MODE`。
- 工具 schema 化。
- 每个工具接入现有权限等级、确认规则、执行日志。
- 默认高风险输出走草稿或只读。

### Stage 8: 自动代聊试验

前置条件：

- Stage 5B 评估体系可用。
- Stage 6B 权限和确认机制可用。
- Stage 7 受控工具循环可用。

初始策略应是信任名单内、私聊、低风险场景先草稿，稳定后再考虑有限自动回复。
