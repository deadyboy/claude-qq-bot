# QQ Bot Command Guide

本文档记录当前 QQ bot 已实现的稳定命令。除特别说明外，私聊可直接发送；群聊中需要 @ bot 或回复 bot 的消息才会处理。

## 通用规则

- 私聊：默认会响应普通聊天和命令。
- 群聊：默认只响应 @ bot 或回复 bot 的消息。
- Owner-only：只有 `.env` 中 `OWNER_QQ_IDS` 配置的主人账号可用。
- Private-only：只能私聊使用，避免在群里暴露隐私、联系人、群号、风格样本或草稿。
- 运行数据默认写入 `data/`，不会提交到 Git。

## 基础帮助与权限

### `/help`

显示基础命令列表。

用法：

```text
/help
```

### `/tools`

查看当前可用工具。普通用户只看到低风险工具；主人会看到管理工具。

别名：

```text
/tools
/工具
工具
工具列表
```

### `/权限`

查看当前用户权限状态，包括是否为主人、是否在信任用户名单中。

别名：

```text
/权限
/owner
权限
我的权限
```

## 普通聊天与会话

### 普通聊天

直接发送自然语言即可。bot 会结合短期会话、机器人 persona、当前用户画像回复。

群聊中必须 @ bot 或回复 bot。

### `/clear`

清空当前会话历史。

权限：

- 私聊：当前用户可清空自己的当前会话。
- 群聊：Owner-only，并需要 `/确认 <id>` 二次确认。

别名：

```text
/clear
清空历史
清除记忆
```

群聊中发送 `/clear` 后会返回确认 ID：

```text
/确认 <id>
/取消 <id>
```

## 确认与取消

### `/确认`

执行待确认操作。当前用于群聊清空、白名单变更、清空风格样本等高风险动作。

权限：Owner-only。

用法：

```text
/确认
/确认 <id>
确认 <id>
/confirm <id>
```

说明：

- 不带 ID 时列出当前用户的待确认操作。
- 待确认操作默认 10 分钟过期。
- 执行结果会写入 `data/action_logs.jsonl`。

### `/取消`

取消待确认操作。

权限：Owner-only。

用法：

```text
/取消
/取消 <id>
取消 <id>
/cancel <id>
```

说明：

- 不带 ID 时列出当前用户的待确认操作。
- 取消结果会写入 `data/action_logs.jsonl`。

## 用户资料记忆

### `记住：...`

显式保存当前用户资料。适合保存称呼、偏好、长期背景等。

用法：

```text
记住：我喜欢简洁直接的回答
记住：称呼=小付
/remember 我是一名程序员
```

### `忘记：...`

删除当前用户资料。

用法：

```text
忘记：偏好
忘记我的资料
清除我的资料
删除我的资料
/forget 偏好
/forget
```

### `我的资料`

查看 bot 记住的当前用户资料。

别名：

```text
我的资料
我的记忆
你记住了什么
/profile
```

### `记忆查询`

搜索当前用户画像中已保存的资料。

用法：

```text
记忆查询 关键词
/memory search 关键词
/memory 查询 关键词
```

### `记忆开关`

控制自动事实抽取。显式 `记住：...` 不受影响。

权限：Owner-only。

用法：

```text
记忆开关
记忆开关 开
记忆开关 关
/memory on
/memory off
```

## 工具命令

### `时间`

查看本机当前时间。

别名：

```text
时间
现在几点
/time
/时间
```

### `计算`

执行安全数学计算。只支持数字、括号、`+ - * / // % **`，不支持函数、变量、导入、属性访问。

用法：

```text
计算：1 + 2 * 3
计算 (10 + 2) / 3
/calc 2 ** 8
/calculate 100 / 4
```

### `待办`

管理当前 QQ 用户自己的待办。

用法：

```text
待办
待办 添加 买牛奶
待办 新增 明天整理资料
待办 完成 1
待办 完成 <待办ID前缀>
/todo
/todo add 买牛奶
/todo done 1
/待办 列表
```

说明：

- `待办` 或 `待办 列表` 查看未完成待办。
- `待办 添加 ...` 新增待办。
- `待办 完成 1` 按列表序号完成待办。
- `待办 完成 <ID>` 按待办 ID 前缀完成待办。

## 信任名单

信任名单是教学影子审核和高风险工具的权限基座。普通聊天兜底只允许主人和信任群；信任私聊对象在未开启 `/教学` 时不会走普通聊天兜底。

权限：Owner-only、Private-only。

白名单新增和删除都会先生成待确认操作，需要再发送 `/确认 <id>` 才会真正写入。

### 查看名单

```text
/白名单
白名单
信任名单
/access
```

### 添加或删除信任用户

```text
/白名单 添加用户 <QQ> [备注]
/白名单 删除用户 <QQ>
```

示例：

```text
/白名单 添加用户 123456789 老朋友
/白名单 删除用户 123456789
```

### 添加或删除信任群

```text
/白名单 添加群 <群号> [备注]
/白名单 删除群 <群号>
```

示例：

```text
/白名单 添加群 987654321 测试群
/白名单 删除群 987654321
```

## 风格画像与草稿

风格画像用于学习主人的说话方式。风格命令当前全部是 Owner-only、Private-only。

### `/风格 查看`

查看当前风格画像摘要，包括语气、长度、表情习惯、样本数、导入来源和蒸馏统计。

```text
/风格 查看
```

### `/风格 设置`

手动设置风格字段。

可设置字段：

- `语气`
- `长度`
- `表情`
- `习惯`
- `避免`

用法：

```text
/风格 设置 语气=自然、简短、像我本人
/风格 设置 习惯=短句；少解释；必要时用一点表情
/风格 设置 避免=编造事实；替主人承诺行动
```

### `/风格 导入`

手动导入一小段主人真实回复样本。这个样本会作为可选示例进入风格 prompt。

用法：

```text
/风格 导入 在的在的，刚看到，我来处理
```

说明：

- 单条样本有长度限制。
- 疑似包含 token、密码、密钥等敏感内容会被拒绝保存。

### `/风格 导入文件`

导入聊天记录文件并生成预览。文件必须先放到：

```text
data/style_profiles/import_inbox/
```

支持格式：

- `.txt`
- `.json`
- `.csv`

用法：

```text
/风格 导入文件 <文件名> 我=<你的昵称或QQ>
```

示例：

```text
/风格 导入文件 最近联系人.txt 我=36
/风格 导入文件 chat.csv 我=你的昵称,你的QQ号
```

说明：

- 第一步只生成预览和 `import_id`。
- 预览会统计解析记录数、你的消息数、跳过敏感消息数、平均长度和常见表达。
- 不会保存对方原话到画像。
- 不会写入普通用户记忆。

### `/风格 确认导入`

确认一次文件导入预览，并把蒸馏摘要写入风格画像。

用法：

```text
/风格 确认导入 <import_id>
```

示例：

```text
/风格 确认导入 20260508_025210_8a717014da
```

### `/风格 蒸馏`

从已保存的直接样本重新生成风格画像摘要。

用法：

```text
/风格 蒸馏
```

说明：

- 当前主要用于直接 `/风格 导入 <sample>` 的小样本重建。
- 批量文件导入已经在确认导入时写入本地蒸馏摘要。

### `/风格 离线蒸馏`

从 QCE JSON 导出目录运行 Stage 5B 离线蒸馏，生成风格摘要和高质量样本索引。

用法：

```text
/风格 离线蒸馏
/风格 离线蒸馏 <QCE_JSON目录>
```

说明：

- Owner-only、Private-only。
- 默认输入目录会从配置的 QCE 导出根目录中选择最近的 JSON 批次。
- 可通过环境变量 `QQBOT_STYLE_QCE_INPUT_DIR`、`QQBOT_STYLE_EXPORT_ROOT`、`QQBOT_STYLE_SELF_UIN` 覆盖默认输入目录、导出根目录和本人 QQ。
- 输出目录默认在对应导出批次的 `distill-runs/` 下。
- 摘要和样本索引不保存聊天正文。
- 样本索引只保存 `source_file_id`、记录序号、消息 ID hash、时间桶、长度、元素类型、质量分等元数据。
- 为本地复现保留 `source_catalog_private.json`，其中只映射源文件，不保存聊天正文。
- 命令执行可能较慢，建议长批次由 Codex 旁路监控 QCE/NapCat 状态。

### `/风格 评估`

查看最近一次 Stage 5B 离线蒸馏的就绪度、样本量、关系来源数量和风险建议。

用法：

```text
/风格 评估
```

说明：

- Owner-only、Private-only。
- 只读取 `evaluation_report.json`。
- 不显示历史上下文、真实回复正文、联系人、群名或 QQ。

### `/风格 关系`

查看关系/场景来源画像摘要，例如私聊/群聊数量、强样本源数量、低证据来源数量和标签分布。

用法：

```text
/风格 关系
```

说明：

- Owner-only、Private-only。
- 只显示 `source_file_id` 和统计标签，不显示联系人、群名、QQ 或聊天正文。

### `/风格 场景`

查看场景画像摘要，例如群聊插话、私聊短回复、显式回复、问句回复等场景分布。

用法：

```text
/风格 场景
```

说明：

- Owner-only、Private-only。
- 只显示场景标签、计数、平均长度和推荐回复形态。
- 不显示历史上下文或真实回复正文。

### `/风格 检索`

用当前对方消息在本地历史索引中检索相似样本。

用法：

```text
/风格 检索 你看下这个安排行不行
```

说明：

- Owner-only、Private-only。
- 会临时读取本地 QCE JSON 计算相似度，但不会返回或保存历史聊天正文。
- 检索使用混合分数：字符重叠、中文关键词、中文意图、私聊/群聊类型、关系来源和样本质量。
- 返回结果只包含 `sample_id/source_file_id`、相似度、质量分、回复长度、上下文数量和分数拆分。

### `/风格 调试`

查看当前消息在 Stage 5B-RAG 链路中的真实检索过程和命中样本原文。

用法：

```text
/风格 调试 你现在忙吗
/风格 调试 这个怎么弄
```

说明：

- Owner-only、Private-only。
- 会显示中文意图识别、当前关系映射、相似度拆分、生成策略、top 历史样本。
- 会显示普通聊天原文，便于检查“对方上下文 -> 主人回复”是否真的像你。
- API key、token、密码、证件号等凭据类文本仍会被跳过或屏蔽。
- 不会把原文写入画像、蒸馏输出、样本索引或审计日志。

### `/风格 原句`

控制真实历史原句 few-shot 是否允许进入模型提示词。

用法：

```text
/风格 原句 状态
/风格 原句 开
/风格 原句 关
```

说明：

- Owner-only、Private-only。
- 默认关闭。
- 开启时会生成待确认操作，需要再发送 `/确认 <id>`。
- 开启后，`/用我的风格回复` 和 `/教学` 候选生成可以把少量经过脱敏的真实历史上下文与主人回复作为 few-shot 发送给模型。
- 不会把 few-shot 原文写入画像、蒸馏输出或审计日志。
- 审计日志只记录 `sample_id/source_file_id`、字数、目标 hash、运行批次等元数据。

### 已退役：`/风格 自动回复` / `/代聊`

旧的 owner-style 自动发送入口已经退役。命令仍会返回说明，但不会开启自动回复，也不会创建确认动作。

用法：

```text
/代聊
/风格 自动回复
```

说明：

- 自动发送入口不可开启。
- 需要观察真实联系人时使用 `/教学 开`：信任用户私聊只生成候选给主人审核，不自动发给对方。
- 需要手动构造回复时使用 `/用我的风格回复：<对方消息>`。

### `/风格 清空样本`

清空手动导入的小样本。

用法：

```text
/风格 清空样本
```

说明：

- 会生成待确认操作，需要再发送 `/确认 <id>`。
- 当前只清空直接样本，不等同于删除所有历史导入来源。

### `/用我的风格回复`

按当前风格画像生成回复草稿，不会代替主人自动发送。若已有 Stage 5B 离线蒸馏结果，会自动接入相似样本索引、关系画像和场景画像的统计元数据来生成草稿。

用法：

```text
/用我的风格回复：你看下这个安排行不行
用我的风格回复：这个怎么回比较自然
风格回复：对方消息
/style draft 对方消息
```

说明：

- 返回草稿正文，并创建一个可评分的教学样本 ID。
- 不会自动发送给对方。
- 会读取 Stage 5B 的相似样本索引、关系/来源标签、场景建议和长度统计，形成“生成闭环 v1”。
- 默认闭环不会把历史聊天原文发送给模型，只使用统计和索引元数据。
- `/风格 原句 开` 经确认后，会允许少量脱敏真实历史原句 few-shot 进入模型提示词，并写审计元数据。
- 生成链路会识别无问号中文问句，例如“你现在忙吗”“在不在”“这个怎么弄”“能不能帮我看下”。
- 相似度过低时不会强行注入 raw few-shot，避免拿无关历史样本误导生成。
- 遇到主人当前状态、位置、是否有空、是否完成某事等未知事实时，会尽量生成不确认事实的自然草稿。

### `/教学`

开启或关闭 owner-style 影子审核模式。开启后，信任用户私聊不会自动回复对方；bot 会把对方消息和最多 8 条候选回复私发给主人，由主人评分、采纳、改写或拒绝。

权限：Owner-only、Private-only。

用法：

```text
/教学 状态
/教学 开
/教学 关
/教学 最近
/教学 复盘
/教学 出题 10
/教学 出题 10 private_short_casual
/教学 纠正
/教学 纠正 最近
/教学 纠正 停用 <纠正ID>
```

反馈命令：

```text
/采纳 <教学ID> <1-8> [原因]
/评分 <教学ID> <1-5> [原因]
/改成 <教学ID> <你会怎么回>
/拒绝 <教学ID> <原因>
```

如果只处理最近一条教学样本，可以省略教学 ID：

```text
/采纳 1
/评分 3 太正式
/改成 我先看下，别等我这边确定
/拒绝 没有理解对方意思
```

说明：

- 教学模式复用现有风格生成、Stage 5B RAG 和 `36.skill` 运行时人格层，不另起一套重复生成逻辑。
- `/教学 出题` 会从 `sft_candidates.jsonl` 抽取文本可归因的高质量历史样本，批量生成审核题；适合快速积累评分和改写反馈。
- 反馈写入 `data/style_profiles/teaching_feedback.jsonl`，用于后续回放评估、rerank 规则和训练样本筛选。
- `/改成` 会同时写入 `data/style_profiles/36_skill/corrections.jsonl`，作为下一次生成主动读取的纠正层。
- 影子审核只对信任用户私聊生效；群聊和非信任用户不会进入该模式。
- 旧 `/代聊` 自动回复已退役；真实联系人观察统一使用 `/教学 开`。

## Legacy Agent Mode
